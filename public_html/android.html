Description of Components of the System:

There are a few components that make up my Android application. To start off, I decided to use the magnetic field sensor to give me the phone’s orientation data. I had started with using the gyroscope sensor before concluding that it was not the best fit in terms of what it’s built for and then tried implementing the gravity sensor. For some unusual reason, the gravity sensor was not feeding back information. Another component that I used was the media player API. I took my five audio clips and moved them into a designated folder within the project package and then called upon each depending on the orientation of the phone that I chose. Another component I found that was great was the runOnUiThread method. It makes sure all the UI actions are in one thread so that the UI is made up of components running in a sequential and continuous flow. The user interface is made up of three text views and three edit texts. I wanted the application to show the information that the gyroscope sensor was getting in real-time so the user can be aware of any possible discrepancies or offsets depending on their location. The user interface shows the x-axis, y-axis, and z-axis. Lastly, another component to the system is the one that controls playing the audio. If audio is already playing, it does nothing. If no audio is playing, then it can go ahead and call the audio for the current orientation registered.
